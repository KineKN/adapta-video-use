# Ad Copy Structure

## Overview

In Adapta ads, the target is not "shorter video". The target is a clean commercial arc.

Default arc:

`Hook -> Lead -> Body -> Proof -> CTA`

## When to Use

- Bruto transcript has multiple attempts
- You need to decide what is real copy vs set noise
- You are unsure whether a line is hook, body, proof, or CTA
- You need to preserve intention while cutting aggressively

Do not use this as a subtitle style guide or visual editing guide.

## Core Pattern

### Hook

The first line that makes the viewer care now.

Examples:

- price shock
- status threat
- curiosity
- authority transfer

### Lead

Reframes the pain or introduces the thesis.

### Body

Main mechanism, offer, or explanation.

### Proof

Why this is credible, safe, or worth attention.

### CTA

Single action close. Usually only one clean CTA should remain.

## Rules

- Preserve the original hook if it is part of the real commercial promise.
- Do not replace a strong hook with a cleaner secondary line just because it cuts better.
- If the ad contains `proof -> restart -> proof -> CTA`, keep one proof and one CTA.
- If the transcript is weak or truncated, allow the final to be more complete than the bruto.
- Preserve concrete selling details before abstract summaries. Named tools, counts, pricing, ranking position, and the exact offer often sell better than generic paraphrase.
- In Adapta ads, identify the house funnel before cutting. The same detail can be optional in one family and mandatory in another.
- If a funnel spends real time building problem pressure, make sure the cut still lands the concrete rescue offer, not just the pressure.

## Corpus Reality

From the local corpus:

- most pairs are cleanup, not rewrites
- some outliers are heavy rebuilds
- some finals are longer than brutos because bruto text is incomplete

See `skills/ad-corpus-study/references/corpus-notes.md` for the detailed interpretation.

## Quick Checklist

- What is the hook?
- What is the thesis?
- What line actually explains the offer?
- What line gives proof?
- Where is the final CTA?
- Is any part duplicated semantically?

## Common Mistakes

- Mistaking a cleaner line for a better opener
- Replacing concrete offer details with generic category language
- Keeping the arc but deleting the mechanism that makes that funnel persuasive
- Keeping two versions of the same proof
- Keeping pre-CTA setup plus final CTA when only one is needed
- Cutting the body so hard that the promise stops making sense
